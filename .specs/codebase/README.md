# Codebase Docs Index

**Projeto:** AppHosp  
**Última revisão:** 2026-04-08

Este diretório concentra a documentação brownfield do AppHosp em arquivos curtos por tema.

## Ordem recomendada de leitura

1. `STACK.md` — visão rápida da stack técnica
2. `STRUCTURE.md` — onde cada responsabilidade fica
3. `ARCHITECTURE.md` — fluxos e padrões estruturais
4. `CONVENTIONS.md` — convenções de código e regras práticas
5. `INTEGRATIONS.md` — Supabase, CDN, fonts e hosting
6. `CONCERNS.md` — dívidas, riscos e áreas frágeis
7. `TESTING.md` — estado atual de testes e lacunas

## Documento mais completo

Para uma visão corrida do sistema inteiro, use `../../BROWNFIELD_MAPPING.md`.

## Manutenção desta documentação

- Atualize contagens e descrições quando a estrutura do projeto mudar.
- Prefira refletir o estado real do código em vez de intenções futuras.
- Registre riscos reais em `CONCERNS.md` e não no README principal.
- Quando houver feature parcial ou roadmap ativo, explicite isso em vez de misturar com o núcleo já produtivo.
